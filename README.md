## Overview

This project implements and benchmarks distributed sparse matrix-vector multiplication (SpMV) using MPI with 1D cyclic partitioning. The implementation uses optimized collective communication via `MPI_Alltoallv` with pre-computed communication patterns. Performance is evaluated through strong scaling experiments on real sparse matrices from the SuiteSparse collection (up to 2.38M rows) and weak scaling experiments on synthetically generated matrices (scaling proportionally with process count). Both evaluations span 1-256 MPI processes, demonstrating the fundamental scalability limits of 1D partitioning for irregular sparse matrices.

## Code Overview

### Input
- **Strong Scaling:**
  - *Format:* Matrix Market (.mtx) sparse matrices from SuiteSparse
  - *Available matrices:* 10K (bcsstk17), 40K (jnlbrng1), 108K (x104), 217K (pwtk), 377K (ML_Laplace), 504K (af_shell9), 1219K (CurlCurl_3), 2380K (CurlCurl_4)
  - *Note:* Matrices have been renamed from their original SuiteSparse name to the format `<size>.mtx` to simplify testing (e.g., `bcsstk17.mtx` в†’ `10K.mtx`). The full mapping is available in `scripts/matrix/matrix_strong_scaling.txt`.

- **Weak Scaling:**
  - *Format:* Synthetic matrices generated by `matrix_generator.cpp` in Matrix Market (.mtx) format
  - *Matrices generated:* 10K, 20K, 40K, 80K, 160K, 320K, 640K, 1280K, 2560K (5 nnz per row, scaling proportionally with process count)

### Output
For each run, the program generates:

**Strong Scaling:**
- `error_strong.log` - Error messages (PBS stderr)
- `output_strong.log` - Standard output (PBS stdout)
- A folder for each matrix containing:
  - `output_<size>K_<procs>p.log` - Per-run log (e.g., `output_2380K_64p.log`)
  - `strong_scaling_results.csv` - Timings with columns: `matrix, type, procs, time, comp, comm, speedup, eff, flops, mem, imb, min, max`

**Weak Scaling:**
- `error_weak.log` - Error messages (PBS stderr)
- `output_weak.log` - Standard output (PBS stdout)
- `weak_scaling_results.csv` - Global CSV with all weak scaling results
- A folder for each matrix containing:
  - `weak_<size>K_<procs>p.log` - Per-run log (e.g., `weak_640K_64p.log`)
  - `weak_scaling_results.csv` - Local CSV copy

## Implementation Details

### Key Functions

#### `struct CSRMatrix` (line 16)
- Local sparse matrix in Compressed Sparse Row format
- Stores `rowptr[]` (row offsets), `col[]` (column indices), `val[]` (nonzero values)
- Tracks both local row count (`rows`) and global row count (`global_rows`)

#### `struct Metrics` (line 23)
- Performance metrics container
- Stores timing breakdown (`spmv_time`, `comm_time`, `comp_time`, `total_time`), FLOPs, memory footprint and local nnz count

#### `bool is_my_row(int global_row, int rank, int size)` (line 32)
- Returns `true` if `global_row` belongs to the current rank
- Implements cyclic ownership: `global_row % size == rank`

#### `int get_owner(int global_col, int size)` (line 33)
- Returns the rank that owns a given global column index
- Used during setup to identify which remote vector elements are needed: `global_col % size`

#### `int global_to_local(int global_idx, int size)` (line 34)
- Converts a global row/column index to the corresponding local index
- Inverse of cyclic distribution: `global_idx / size`

#### `CSRMatrix read_matrix_distributed(const string& file, int rank, int size)` (line 39)
- Each rank independently reads the full Matrix Market file
- Retains only rows belonging to itself (via `is_my_row`)
- Converts the filtered COO entries directly to local CSR format

#### `class OptimizedSpMV` (line 77)
- Main class that encapsulates the distributed SpMV logic
- Stores all communication buffers and pre-computed index mappings as private members

#### `OptimizedSpMV::setup()` (line 95)
- One-time initialization of the communication pattern
- Identifies all remote vector elements needed by scanning local column indices
- Exchanges metadata with other ranks via `MPI_Alltoall` (counts) and `MPI_Alltoallv` (indices)
- Builds `x_is_remote[]` and `x_idx[]` arrays: for each nonzero, pre-computes whether the corresponding `x` element is local or remote and its position in the appropriate buffer

#### `OptimizedSpMV::run(vector<double>& y, double& comm_t, double& comp_t)` (line 158)
- Executes one SpMV iteration in two timed phases:
  1. **Communication phase:** packs local `x` values into the send buffer, then calls `MPI_Alltoallv` to exchange remote elements with all other ranks
  2. **Computation phase:** iterates over local CSR rows, accumulating the dot product using either `x_local[]` or `recv_buf[]` based on the pre-computed `x_is_remote[]` flag
- Accumulates `comm_t` and `comp_t` separately for breakdown analysis

#### `int main(int argc, char** argv)` (line 186)
- Orchestrates the full execution pipeline:
  1. Initializes MPI and loads the matrix via `read_matrix_distributed`
  2. Runs `setup()` once to pre-compute communication patterns
  3. Performs 1 warmup iteration (not timed)
  4. Runs 50 benchmark iterations, averaging total/comm/comp times
  5. Rank 0 prints results to stdout and appends a row to the CSV log file

```bash
module load gcc91
module load mpich-3.2.1--gcc-9.1.0
```

```bash
g++ -O3 -std=c++11 -o matrix_generator matrix_generator.cpp
./matrix_generator batch
```
> **Note:** To modify the generated matrices, edit the `configs` vector at lines 101вЂ“111 in `matrix_generator.cpp`.

### Main Code
```bash
mpicxx -O3 -std=c++11 -o code code.cpp
```

### Cluster Execution

**Strong Scaling:**
```bash
qsub strong_scaling.pbs
```
Before submitting, edit `strong_scaling.pbs`:
- Line 3: `select`, `ncpus`, `mpiprocs`, `mem` (see `parameters_strong_scaling.txt`)
- Line 4: `walltime`
- Line 5: `short_cpuQ` / `long_cpuQ`
- Lines 69вЂ“71: matrices to test (e.g., `"2380K.mtx"`)
- Line 74: process counts array (e.g., `1 2 4 8 16 32 64 128 256`)

**Weak Scaling:**
```bash
qsub weak_scaling.pbs
```
Before submitting, edit `weak_scaling.pbs`:
- Line 3: `select`, `ncpus`, `mpiprocs`, `mem` (see `parameters_weak_scaling.txt`)
- Line 4: `walltime`
- Line 5: `short_cpuQ` / `long_cpuQ`
- Lines 33вЂ“44: process counts and corresponding matrices

> **Note:** `code.cpp`, the compiled executable `code`, the `.pbs` script, and all `.mtx` matrix files must be in the same working directory. For clarity, use separate directories for strong and weak scaling runs to avoid naming conflicts between synthetic and SuiteSparse matrices of the same size.
